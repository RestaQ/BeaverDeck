# BeaverDeck 1.3.0

`BeaverDeck 1.3.0` is a feature and UX release focused on day-2 operations, authentication, chart cleanup, and production hardening.

## Highlights

- added first-run bootstrap initialization with a one-time token from application logs
- added Google OAuth and generic OpenID Connect login flows with group-to-role mapping
- improved cluster operations UI for workloads, RBAC resources, manifests, logs, and exec sessions
- added lightweight update-check support for reporting the latest available BeaverDeck version
- refreshed dependencies and build toolchain for security and maintainability

## Added

- first-run initialization flow:
  - bootstrap token is written to application logs
  - admin password is set from the UI
  - no pre-created Kubernetes secret is required
- Google OAuth authentication with Google Workspace group-to-role mapping
- generic OpenID Connect authentication with group-to-role mapping
- session revocation for administrators
- local-user password reset
- theme selection:
  - auto
  - dark
  - light
- cluster version visibility in the UI profile and login screen
- RBAC views for:
  - ClusterRoles
  - ClusterRoleBindings
  - Roles
  - RoleBindings
  - ServiceAccounts
- workloads support for:
  - Jobs
  - CronJobs
- search support on major resource pages, including workloads, services, ingresses, config maps, secrets, PVCs, PVs, and service accounts
- `/healthz` as a dedicated public health endpoint for Kubernetes probes

## Changed

- reworked workload actions:
  - scale moved into row actions
  - StatefulSet scaling is supported
- improved manifest editing flow:
  - apply and dry-run now use concise success/error notifications instead of raw JSON blocks
- improved lower dock behavior:
  - exec, logs, and manifest/edit panes can be resized more predictably
  - dock is hidden on Audit and Admin pages without losing open tabs
- manifest and edit tabs now reuse an already open tab for the same resource instead of opening duplicates
- lower dock tabs now truncate long labels and show the full title on hover
- login and branding screens were visually refreshed and aligned with the BeaverDeck logo palette
- default chart ingress configuration was simplified from an ingress list to a single standard ingress object

## Fixed

- fixed pod exec behavior for minimal and distroless containers:
  - shell fallback is cleaner
  - failed exec tabs become effectively read-only
- fixed log refresh and follow-tail behavior to stay at the end of the log
- fixed edit/apply manifest handling for workloads missing `apiVersion` or `kind`
- fixed readiness/liveness probe authorization issues by serving health checks without authentication
- fixed duplicate delete controls and inconsistent actions on resource pages
- fixed layout issues in Insights, User Management, Roles, Cluster Roles, and dock tabs
- fixed namespace selection persistence and page persistence across browser refreshes
- fixed Google and OIDC configuration test behavior

## Security

- refreshed Go runtime baseline and core Go dependencies
- upgraded `golang.org/x/oauth2`
- refreshed Kubernetes client dependencies
- refreshed frontend build dependencies
- local passwords are stored as hashes instead of plaintext

## Deployment Notes

- app version: `1.3.0`
- Helm chart version: `2.0.0`
- breaking chart change:
  - `ingresses` was replaced by a single `ingress` object
- persistence is strongly recommended for production deployments so BeaverDeck does not lose users, roles, audit history, bootstrap state, and other stored settings
