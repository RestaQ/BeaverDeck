# BeaverDeck 1.3.1

`BeaverDeck 1.3.1` is a focused UX and usability release for pod operations and log inspection.

## Highlights

- added pod multi-select with batch evict and delete actions
- split pod row actions into compact direct buttons for logs and exec plus a menu for the remaining actions
- improved pod logs with on-demand older-log loading when scrolling to the top
- cleaned up log rendering and table layout issues in the pods view

## Added

- pod selection checkboxes are now always visible in the Pods table
- select-all checkbox for visible pods in the Pods header
- automatic batch-selection mode:
  - batch mode turns on as soon as at least one pod is selected
  - batch mode turns off automatically when all pod checkboxes are cleared
- batch pod actions for:
  - evict
  - delete
- log history expansion when scrolling to the top of a pod or workload log tab
- explicit log state hints in the lower dock:
  - `Loading older logs...`
  - `Beginning of log reached.`

## Changed

- pod row actions were reorganized:
  - `Logs` and `Exec` are now compact icon buttons
  - `Manifest`, `Evict`, and `Delete` stay in the row action menu
- pod action controls remain visible during multi-select and become disabled instead of disappearing
- pod selection summary and batch actions are shown inline in the pods toolbar only while something is selected
- pod action buttons were reduced in size to keep table rows compact
- log tabs now use a single continuous scroll surface without the extra inner framed block

## Fixed

- fixed truncated pod logs by loading older content instead of stopping at the initial tail buffer
- fixed log scroll-position jumps while prepending older log lines
- fixed invalid log viewer markup that could cause inconsistent rendering
- fixed a visual frame artifact at the beginning of log output
- fixed pod warnings so `Succeeded` pods no longer show a false warning indicator beside readiness
- fixed pods table header and actions column alignment issues
- fixed the actions column border/step artifact caused by flex layout on table cells

## Deployment Notes

- app version: `1.3.1`
- Helm chart version: `2.0.1`
- no chart breaking changes in this release
